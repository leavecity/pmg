# Governance Model

PMG governance defines how project knowledge moves through the system.

## Modules

- Memory Curator: extracts durable knowledge and proposes updates.
- Architecture Guardian: detects architectural drift and layer violations.
- Technical Debt Auditor: records compromises and workarounds.
- Dependency Governor: reviews dependency additions and overlap.
- Security Reviewer: checks common security risks.
- Performance Reviewer: identifies expensive or scaling-sensitive behavior.

These modules are represented as files and skills in the MVP. Future versions may add executable checks, but the first release keeps governance transparent and reviewable.
