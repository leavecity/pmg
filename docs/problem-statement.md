# Problem Statement

AI coding agents repeatedly lose project continuity. The loss is especially painful in medium and large repositories where useful work depends on architecture history, accepted patterns, rejected approaches, dependency constraints, localization rules, state ownership, and known pitfalls.

The problem is not only storage. A repository can store many Markdown files and still fail to answer:

- What is confirmed knowledge?
- What is only inferred?
- What has become stale?
- What conflicts with previous decisions?
- What should an agent load for this task?
- What should be promoted into long-term memory?
- What should remain pending or be archived?

PMG addresses the governance gap between raw project notes and reliable agent context.
