# PMG Constitution for This Repository

This repository implements Project Memory Governance.

## Rules

- Keep the MVP repository-native, file-based, and vendor-neutral.
- Keep default templates readable before adding automation.
- Keep CLI commands deterministic and testable.
- Do not add a database, vector store, or cloud service to v1.
- Treat context assembly as the core product behavior.

## Memory status policy

Confirmed memory in this repository must be backed by project docs, code, tests, or explicit maintainer approval.

Pending memory belongs in `.pmg/memory/pending.md` until reviewed.
